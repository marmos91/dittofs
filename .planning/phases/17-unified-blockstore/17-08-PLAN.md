<objective>
Ship the one-shot legacy-`.blk` → CAS migration as an offline `dfs migrate-to-cas` cobra subcommand backed by a library in `pkg/blockstore/migrate/migrate_to_cas.go`. Idempotent via per-share journal; supports `--dry-run`, `--share`, `--json`; writes `.cas-migrated-v1` sentinel atomically on completion. Refuses to run while server is up.

Purpose: Plan 09's boot guard will refuse to start `dfs` until the sentinel exists. This plan delivers the only legitimate way to produce that sentinel. Without it, every operator with a pre-v0.16 install is bricked.

Output: Operational migration path from legacy to CAS, with crash safety + dry-run preview + machine-readable progress.
</objective>

<interfaces>
Library public surface (post-this-plan):

  // MigrateShareToCAS migrates a single share's legacy .blk tree to CAS.
  // Idempotent: resumes from per-share journal on restart.
  // dryRun = true: walks + reports, no disk writes.
  // progressFn invoked ~1Hz with cumulative stats; nil to suppress.
  func MigrateShareToCAS(
      ctx context.Context,
      shareDir string,                 // <storage_dir>/<share>/
      bs blockstore.BlockStore,        // destination CAS store
      meta MetadataAdapter,            // interface for FileAttr rebuild (minimal — see below)
      opts MigrationOpts,
  ) (MigrationResult, error)

  type MigrationOpts struct {
      DryRun      bool
      Progress    func(MigrationStats)
      ChunkerConf chunker.Config        // FastCDC params; mirror production defaults
      JournalPath string                // override; default = shareDir + "/.dittofs-migrate-to-cas.state"
  }

  type MigrationStats struct {
      FilesDone, BytesDone, DedupHits int64
      FilesPerSec, MiBPerSec, ETASec  float64
  }

  type MigrationResult struct {
      Stats         MigrationStats
      EstDedupRatio float64               // dry-run only
      Duration      time.Duration
  }

  // MetadataAdapter — a thin wrapper around metadata.MetadataStore (see pkg/metadata/store.go)
  // that the migration tool uses to enumerate legacy files + atomically commit each file's
  // new CAS-shaped Blocks manifest.
  //
  // Implementation detail (kept inside the CLI subcommand, NOT this library): wraps an
  // open *metadata.Store via metadata.OpenStore(cfg.Metadata) in offline mode and delegates
  // to existing primitives:
  //   - ListLegacyFiles → uses migrate.WalkShareFiles(ctx, mds, shareName, ...) (existing
  //     helper, see pkg/blockstore/migrate/walk.go) and filters to files whose
  //     FileAttr.BlockLayout == metadata.BlockLayoutLegacy OR FileAttr.Blocks is empty +
  //     FileAttr.Size > 0 (pre-Phase-12 schema variant — files predating the Blocks field
  //     have nil Blocks but non-zero Size).
  //   - UpdateFileBlocks → reads the existing *metadata.File via mds.GetFile(handle),
  //     sets file.Blocks = newBlocks + file.BlockLayout = BlockLayoutCASOnly + clears
  //     file.PayloadID's legacy semantics (NOT the field itself — PayloadID is still the
  //     identity), then commits via mds.PutFile(ctx, file). PutFile IS the atomicity
  //     boundary per pkg/metadata/store.go line 37-40 — backends (badger, postgres) implement
  //     it transactionally per the storetest/ conformance suite.
  type MetadataAdapter interface {
      // ListLegacyFiles enumerates files needing migration. A file is "legacy" when
      // EITHER (a) its FileAttr.BlockLayout == metadata.BlockLayoutLegacy (post-Phase-14
      // schema — explicit gate), OR (b) FileAttr.Blocks is nil/empty AND FileAttr.Size > 0
      // AND FileAttr.Type == FileTypeRegular (pre-Phase-12 schema — files predate the
      // Blocks field). The adapter merges both detection paths and returns the union.
      ListLegacyFiles(ctx context.Context) ([]LegacyFileInfo, error)

      // UpdateFileBlocks atomically:
      //   1. Loads the file by handle (mds.GetFile)
      //   2. Sets file.Blocks = blocks (slice of BlockRef{Hash, Size, Offset})
      //   3. Sets file.BlockLayout = metadata.BlockLayoutCASOnly
      //   4. Commits via mds.PutFile(ctx, file)
      // Backends implement step 4 transactionally — verified by pkg/metadata/storetest/ conformance.
      UpdateFileBlocks(ctx context.Context, handle metadata.FileHandle, blocks []blockstore.BlockRef) error
  }

  // LegacyFileInfo carries the minimum projection the migration library needs to
  // process one file. The Handle (NOT a string FileID) is the metadata-store-native
  // identifier; the migration library passes it back to UpdateFileBlocks.
  type LegacyFileInfo struct {
      Handle    metadata.FileHandle   // opaque per metadata-store contract; routes back to the same row
      Path      string                // share-relative path (for log/journal display)
      PayloadID metadata.PayloadID    // legacy block path root: <share_dir>/<PayloadID>/<idx>.blk
      Size      int64                 // FileAttr.Size; total bytes to chunk
      BlockSize uint32                // legacy fixed block size (from share config or FileAttr); offset math during the walk
  }

Subcommand:

  $ dfs migrate-to-cas [flags]
    --storage-dir string   storage root (default ~/.config/dfs/storage)
    --share string         scope to one share (default: all)
    --dry-run              report counts/dedup ratio/ETA; no writes
    --json                 emit one JSON line per second to stdout
    --config string        config file (passthrough)

Exit codes:
  0 = success
  1 = generic error (PID guard tripped, config load failure, IO error)
  2 = migration failed mid-flight (journal preserved; describe resume to stderr)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → migration tool | Operator runs `dfs migrate-to-cas` against a stopped server's storage dir; tool reads existing `.blk` files (trusted local FS) and writes to the configured BlockStore. |
| tool → destination BlockStore | Tool calls `bs.Put(hash, data)`. The hash is computed locally by the tool; the destination MAY independently verify (s3 backend's `x-amz-meta-content-hash` defense-in-depth survives Plan 03). |
| tool → metadata store | `MetadataAdapter.UpdateFileBlocks` is a privileged write that cuts a file over from legacy to CAS layout. Metadata-store conformance suite (Phase 12 era) validates atomicity. |
| tool → sentinel file | The `.cas-migrated-v1` file gates server boot (Plan 09). Cannot be hand-written safely — operators may try, but the embedded JSON `ToolVersion` is the audit signal. |

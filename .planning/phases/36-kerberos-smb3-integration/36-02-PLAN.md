**1. Extend internal/adapter/smb/auth/spnego.go with MIC helpers:**

```go
// ComputeMechListMIC computes a GSS-API MIC over the SPNEGO mechList for downgrade protection.
// Per RFC 4178, the MIC protects the DER-encoded SEQUENCE OF OIDs from the original NegTokenInit.
// Uses Kerberos GSS-API MIC token format with key usage 25 (acceptor sign) per RFC 4121.
func ComputeMechListMIC(sessionKey types.EncryptionKey, mechListBytes []byte) ([]byte, error)

// VerifyMechListMIC verifies a client-sent SPNEGO mechListMIC.
// Returns nil if the MIC is valid, error if verification fails.
func VerifyMechListMIC(sessionKey types.EncryptionKey, mechListBytes []byte, micBytes []byte) error
```

Use gokrb5's `gssapi.MICToken` with `SetChecksum(sessionKey, 25)` for computation and `Verify()` for verification.

Also add `BuildAcceptCompleteWithMIC(mech, mechToken, mechListMIC)` that builds SPNEGO NegTokenResp including the optional MIC field. The existing `BuildAcceptComplete` does not support the MIC field.

Note: MIC is computed over the mechList bytes, which is the DER-encoded SEQUENCE OF OIDs from the client's NegTokenInit. The `ParsedToken` should expose `MechListBytes` (raw DER bytes of the mechTypes field) for this purpose. Add a `MechListBytes []byte` field to `ParsedToken` if not present, populated during parsing by capturing the raw mechTypes before unmarshaling.

**2. Create pkg/auth/kerberos/identity.go for principal-to-username resolution:**

```go
// IdentityConfig configures how Kerberos principals are mapped to DittoFS usernames.
type IdentityConfig struct {
    // StripRealm controls whether the realm is stripped from principal names.
    // Default: true ("alice@REALM" -> "alice")
    StripRealm bool

    // ExplicitMappings maps full Kerberos principals to DittoFS usernames.
    // These take precedence over the strip-realm default.
    // Example: {"admin@CORP.COM": "superadmin", "svc/host@CORP.COM": "service-account"}
    ExplicitMappings map[string]string
}

// ResolvePrincipal resolves a Kerberos principal to a DittoFS username.
// Lookup order:
//   1. Explicit mapping for "principal@realm" (if configured)
//   2. Strip realm: "alice@REALM" -> "alice" (if StripRealm is true, which is default)
//   3. Strip service prefix: "svc/host" -> "svc" (for service principals)
func ResolvePrincipal(principalName, realm string, config *IdentityConfig) string
```

Default config: `StripRealm: true, ExplicitMappings: nil`. This replaces the inline username extraction in the existing `handleKerberosAuth()`.

The new `handleKerberosAuth(ctx *SMBHandlerContext, mechToken []byte, parsedToken *auth.ParsedToken)` should:

a) **Check KerberosService** (not just KerberosProvider):
   - Add `KerberosService *kerberos.KerberosService` field to Handler struct (from `internal/auth/kerberos`)
   - If KerberosService is nil, return STATUS_LOGON_FAILURE

b) **Derive CIFS SPN**:
   - Auto-derive from `KerberosProvider.ServicePrincipal()`: `nfs/host@REALM` -> `cifs/host@REALM`
   - Check Handler for optional `SMBServicePrincipal` override field (from control plane settings)
   - If override is set and non-empty, use it instead of auto-derived

c) **Authenticate via shared service**:
   ```go
   authResult, err := h.KerberosService.Authenticate(mechToken, smbPrincipal)
   ```

d) **Normalize session key to 16 bytes**:
   ```go
   func normalizeSessionKey(key []byte) []byte {
       normalized := make([]byte, 16)
       copy(normalized, key) // truncates >16, zero-pads <16
       return normalized
   }
   sessionKey := normalizeSessionKey(authResult.SessionKey.KeyValue)
   ```

e) **Resolve principal to username**:
   ```go
   username := kerberos.ResolvePrincipal(authResult.Principal, authResult.Realm, h.IdentityConfig)
   ```

f) **Look up user in control plane** (same pattern as existing code):
   - GetUser from UserStore, return STATUS_LOGON_FAILURE if not found or disabled
   - Per user decision: valid Kerberos ticket but unknown principal = hard failure (not guest)

g) **Create session and configure signing**:
   ```go
   sess := h.CreateSessionWithUser(sessionID, ctx.ClientAddr, user, authResult.Realm)
   h.configureSessionSigningWithKey(sess, sessionKey, ctx)
   ```

h) **Build SPNEGO response with AP-REP and MIC**:
   - Determine client's Kerberos OID from `parsedToken` (MS OID vs standard OID)
   - Use `authResult.APRepToken` (raw AP-REP from shared service) as SPNEGO mechToken
   - If `parsedToken.MechListMIC` is present, verify it via `auth.VerifyMechListMIC()`

   - Build response: `auth.BuildAcceptCompleteWithMIC(clientKerberosOID, apRepToken, serverMIC)`

i) **Return authenticated response** via `buildAuthenticatedResponse()` with appropriate encryptData flag

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| engine.loadByHash → local.Get | bytes returned by local.Get flow into the engine Cache LRU and are returned to readers on subsequent Cache hits |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-02-01 | T (Tampering) | Cache LRU entry contents | mitigate | local.Get returns a fresh allocation per Plan 16-01 D-03 contract; Cache.Put copies into the LRU slot; aliasing-defense test in 16-01 Task 2 enforces non-aliasing. |
| T-16-02-02 | I (Information Disclosure) | removal of fileBlockStore.GetByHash gate in loadByHash | accept | The prior gate translated payloadID → ContentHash for legacy FileBlock plumbing; it was not an authorization boundary. The hash is the authoritative key; if a caller has the hash, they had legitimate metadata access to obtain it. No new disclosure surface. |
| T-16-02-03 | D (Denial of Service) | unbounded buffer alloc on miss | mitigate | Chunk size is bounded by FastCDC params (max=16 MiB per Phase 10); Cache budget caps total memory; loadByHash allocates at most one chunk per call and the Cache's LRU eviction reclaims old slots. |
| T-16-02-04 | R (Repudiation) / Observability | removal of mmap-error fallthrough log lines | accept | The deleted fallthrough was an unexpected-error path that, in practice, never fired in production telemetry (warm cache; mmap was already the steady-state path). Phase 16's simplified loadByHash returns local.Get's error directly, preserving observability through the existing Cache miss-error counter. |
</threat_model>

<objective>
Land the read-side complement of PR-A: every CAS GET is BLAKE3-verified end-to-end (INV-06) via a streaming `io.Reader` wrapper, and the engine routes per-block reads between the new CAS path and the legacy `{payloadID}/block-{N}` path based on metadata key shape (D-21) — no S3 trial-and-error, no doubled GET cost. Add the cache-key bifurcation that lets the dual-read window co-exist (D-22) until Phase 14 closes the legacy path.

Purpose: This is the safety net behind CAS — without verification, a silent S3 corruption would propagate corrupt bytes to NFS/SMB clients. The dual-read resolver is what keeps existing files readable through Phase 14.
Output: Streaming verifier, header pre-check, dual-read engine resolver, cache key bifurcation, unit tests for each.
</objective>

       ```go
       package s3

       import (
           "errors"
           "fmt"
           "io"

           "lukechampine.com/blake3"
           "github.com/marmos91/dittofs/pkg/blockstore"  // adjust import path to actual module

       // verifyingReader wraps an io.ReadCloser and feeds every byte through a
       // BLAKE3 hasher. On Close, the accumulated hash is compared to expected;
       // mismatch returns ErrCASContentMismatch. The wrapper is single-use and
       // not goroutine-safe.
       type verifyingReader struct {
           src      io.ReadCloser
           hasher   *blake3.Hasher
           expected blockstore.ContentHash
           done     bool
           closeErr error
       }

       func newVerifyingReader(src io.ReadCloser, expected blockstore.ContentHash) *verifyingReader {
           return &verifyingReader{src: src, hasher: blake3.New(blockstore.HashSize, nil), expected: expected}
       }

       func (v *verifyingReader) Read(p []byte) (int, error) {
           n, err := v.src.Read(p)
           if n > 0 {
               // Hasher.Write never errors per docs.
               _, _ = v.hasher.Write(p[:n])
           }
           if errors.Is(err, io.EOF) {
               v.done = true
               if mismatchErr := v.checkHash(); mismatchErr != nil {
                   return n, mismatchErr
               }
           }
           return n, err
       }

       func (v *verifyingReader) Close() error {
           v.closeErr = v.src.Close()
           if !v.done {
               // Caller didn't read to EOF — we cannot verify. Surface as mismatch
               // (treat as untrusted) rather than silently returning OK.
               return fmt.Errorf("%w: stream closed before EOF", blockstore.ErrCASContentMismatch)
           }
           return v.closeErr
       }

       func (v *verifyingReader) checkHash() error {
           var got blockstore.ContentHash
           sum := v.hasher.Sum(nil)
           copy(got[:], sum)
           if got != v.expected {
               return fmt.Errorf("%w: got %s, want %s", blockstore.ErrCASContentMismatch, got.CASKey(), v.expected.CASKey())
           }
           return nil
       }
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| S3 → engine | Bytes returned from a remote object store; treated as untrusted until BLAKE3 verified |
| Cache → engine | Bytes served from in-memory cache; trusted because they were verified on insert |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-B-01 | Tampering | S3 silent corruption | mitigate | Streaming BLAKE3 verifier (D-18) recomputes over the body bytes; mismatch returns ErrCASContentMismatch and the buffer is discarded — bad bytes never reach the caller. |
| T-11-B-02 | Tampering | Header-only verification | mitigate | D-19 header pre-check is fast-fail only; the streaming recompute over the body is authoritative. Header alone is insufficient (would trust S3 metadata layer). |

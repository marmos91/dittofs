<objective>
Wire Opt 3 (direct-to-Cache on chunk completion) in two parts:

1. Producer side — `pkg/blockstore/local/fs/chunkstore.go::lruTouch` (line 104) invokes `bc.onChunkComplete(hash, data, path)` exactly once per successful touch, post-disk-store. Nil-safe.
2. Consumer side — `pkg/blockstore/engine/engine.go` (where FSStore is constructed) passes `OnChunkComplete: bs.cache.Put` via `FSStoreOptions`. This binds the callback to the engine's RAM-only Cache.

Purpose: Eliminate the "wrote then read" disk hop on NFS COMMIT-then-READ. Every successful chunk write leaves a warm Cache entry; subsequent reads of that hash skip the disk fetch.

Output: chunkstore.lruTouch fires the callback; engine wires it to Cache.Put; nil-callback path is a silent no-op (Plan 04 + Plan 07 together establish D-12 nil-safety contract end-to-end).
</objective>

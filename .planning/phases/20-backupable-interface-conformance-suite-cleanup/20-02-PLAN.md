    BackupableStoreFactory type: func(t *testing.T) metadata.MetadataStore (same signature as StoreFactory per D-14; the suite type-asserts to Backupable inside)

    RunBackupConformanceSuite(t *testing.T, factory BackupableStoreFactory):
    - t.Helper()
    - At top: create store via factory, type-assert b, ok := store.(metadata.Backupable); if !ok t.Fatal("factory must return a store implementing Backupable") (use t.Fatal not t.Skip per RESEARCH.md open question 2 -- factory explicitly opts in)
    - Dispatch 5 subtests via t.Run

    Subtest 1: "RoundTrip"
    - Create source store via factory, populate with a share + 2 files (use createTestShare, createTestFile helpers from suite.go) with BlockRef hashes
    - Call b.Backup(ctx, &buf) to get stream + HashSet
    - Create destination store via factory (fresh, empty)
    - Call destB.Restore(ctx, &buf)
    - Verify files and shares exist in destination with identical data (GetFile, compare PayloadID, Blocks, Mode, Size)
    - Verify HashSet.Len() matches expected unique hash count

    Subtest 2: "ConcurrentWriter" (per D-16)
    - Create store, populate with initial data
    - Start Backup in a goroutine; while backup runs, write new files in a separate goroutine using sync.WaitGroup
    - Restore the backup into a fresh store
    - Verify restored data matches ONLY the initial data (new files must NOT appear)
    - This proves snapshot isolation per ENG-02 "same atomic snapshot transaction"

    Subtest 3: "Corruption" (per D-15, 3 scenarios)
    - Scenario A (truncated): backup to buffer, truncate to half length, attempt Restore, verify errors.Is(err, metadata.ErrRestoreCorrupt)
    - Scenario B (bit-flip): backup to buffer, flip one bit in the middle of the payload region, attempt Restore, verify errors.Is(err, metadata.ErrRestoreCorrupt)
    - Scenario C (wrong engine tag): backup to buffer, manually replace engine tag bytes in the header with a different string, attempt Restore; the driver should detect the mismatch. If the driver delegates engine-tag validation to the envelope, it returns ErrSchemaVersionMismatch or ErrRestoreCorrupt -- either is acceptable

    Subtest 4: "NonEmptyDest" (per ENG-03)
    - Create store, populate with a share + 1 file
    - Backup to buffer
    - Create destination store, populate with a DIFFERENT share + file (making it non-empty)
    - Call Restore, verify errors.Is(err, metadata.ErrRestoreDestinationNotEmpty)

    Subtest 5: "HashSetCorrectness" (per D-17, 2 scenarios)
    - Scenario A (exact match): create store with N files whose BlockRefs use K unique hashes (some shared across files for dedup). Backup, get HashSet. Manually walk all files via GetFile, collect all unique BlockRef hashes into a reference set. Verify HashSet contents match exactly (same hashes, same count)
    - Scenario B (dedup): create two files that share the same BlockRef hash. Backup, verify HashSet.Len() == 1 (not 2). Verify HashSet.Contains returns true for the shared hash

    Use test helper functions following createTestShare/createTestFile patterns from suite.go. Use t.Helper() on helpers. Use t.Fatalf for all assertions (no testify).

    The suite must compile cleanly even though no driver implements Backupable yet -- it only references the Backupable interface and MetadataStore interface via the factory. Verify with go build ./pkg/metadata/storetest/...

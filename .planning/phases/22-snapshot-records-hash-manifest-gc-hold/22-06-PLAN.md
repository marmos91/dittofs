      5. Call `r.GetMetadataStoreForShare("data").(metadata.Backupable).Backup(ctx, &buf)` to produce a HashSet representing the "snapshot-at-T0" view of the metadata. To force H_snap into the snapshot manifest without it being in the live FileBlock metadata, the test BYPASSES Backup's "real" hash extraction by constructing a HashSet directly via `blockstore.NewHashSet(0)` + `Add(H1); Add(H_snap)`. This intentional divergence simulates the canonical "snapshot taken at time T0 then file deleted at T1" scenario — without it the test would not distinguish between "live FileBlock hash" and "snapshot-held hash".

         Alternative if the executor prefers a more faithful round-trip: keep both H_snap and H_orphan as referenced FileBlocks, run Backup to extract them naturally into a HashSet, then DELETE H_snap from metadata (via memory store DeleteFile) before running GC pass #1.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator -> Resetable.Reset | Only `Runtime.RestoreSnapshot` (P24-03) is expected to call Reset in production; no protocol handler does. The boundary is the type-assertion site in `Runtime.RestoreSnapshot`. |
| Reset -> backend storage | Reset is a destructive primitive that bypasses Backupable.Restore's `ErrRestoreDestinationNotEmpty` guard. The barrier protecting it is the share-disabled precondition (D-24-01) enforced in P24-03. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01-01 | Tampering | Resetable callers | mitigate | Resetable is documented as an optional capability invoked only by Runtime.RestoreSnapshot after share.Enabled=false is verified (D-24-01); doc-comment on the interface warns against direct invocation. |
| T-24-01-02 | Denial of Service | Postgres Reset under load | accept | Postgres Reset acquires a single connection via `s.pool.Acquire` and runs TRUNCATE ... CASCADE inside REPEATABLE READ. While the tx holds, concurrent writers block. This is the documented expectation since Reset only runs during operator-initiated restore on a disabled share — no concurrent writers should exist. No mitigation beyond doc-comment. |
| T-24-01-03 | Information Disclosure | Reset error messages | mitigate | Error messages wrap with short prefixes ("badger reset:", "reset:") and inner backend error — no share path, snapshot ID, or operator identity leaked. Operator-facing only. |
| T-24-01-04 | Tampering | Postgres migrations adding new tables without updating backupTables | mitigate | TestBackupTablesCoversAllMigrations (Task 2) is the CI guard; any new CREATE TABLE statement that lands without a backupTables update fails the test. |
| T-24-01-05 | Repudiation | Reset action | accept | No audit log emitted at this layer; structured slog is added by P24-03 at the orchestration boundary. Reset is a primitive, not an operation. |
| T-24-01-SC | Tampering | npm/pip/cargo installs | n/a | No new package dependencies; reuses existing `github.com/dgraph-io/badger/v4` and `github.com/jackc/pgx/v5/pgconn` already in go.mod. |
</threat_model>

**doc.go**: Package documentation explaining this is the shared Kerberos service layer used by both NFS RPCSEC_GSS and SMB SESSION_SETUP.

**replay.go**: Implement `ReplayCache` as an in-memory, TTL-based cache:
- Key: `(principal, ctime, cusec, servicePrincipal)` -- 4-tuple for cross-protocol replay detection
- `NewReplayCache(ttl time.Duration)` constructor, default TTL = 5 minutes
- `Check(principal string, ctime time.Time, cusec int, servicePrincipal string) bool` -- returns true if replay detected, false if new (and records it)
- Uses `sync.Map` for concurrent access
- Lazy expiry: clean expired entries on Check calls (every N calls or when entry is accessed)
- Thread-safe for concurrent NFS and SMB authentication

**service.go**: Implement `KerberosService`:
```go
type AuthResult struct {
    Principal    string
    Realm        string
    SessionKey   types.EncryptionKey  // subkey preferred over ticket session key
    APReq        messages.APReq       // preserved for AP-REP construction
    APRepToken   []byte               // raw AP-REP bytes (not GSS-wrapped)
}

type KerberosService struct {
    provider    *kerberos.Provider
    replayCache *ReplayCache
}

func NewKerberosService(provider *kerberos.Provider) *KerberosService
func (s *KerberosService) Authenticate(apReqBytes []byte, servicePrincipal string) (*AuthResult, error)
func (s *KerberosService) BuildMutualAuth(apReq *messages.APReq, sessionKey types.EncryptionKey) ([]byte, error)
```

Extract the core logic from `internal/adapter/nfs/rpc/gss/framework.go`:
- `Authenticate()`: Unmarshal AP-REQ, build service.Settings from provider keytab, call `service.VerifyAPREQ()`, check replay cache, decrypt authenticator, extract subkey (prefer over ticket key), return AuthResult
- `BuildMutualAuth()`: Copy the proven `buildAPRep()` implementation from NFS GSS -- EncAPRepPart with ctime/cusec from authenticator, optional subkey, encrypt with session key (key usage 12), marshal as APPLICATION 15. Return raw AP-REP bytes (NOT GSS-wrapped -- callers handle their own framing: NFS adds GSS-API wrapper, SMB uses SPNEGO).

Note: Do NOT wrap in GSS-API token in BuildMutualAuth. NFS GSS will add its own 0x60 + OID + 0x0200 wrapper. SMB will pass raw AP-REP to SPNEGO. This is different from the existing buildAPRep() which includes GSS wrapping -- the shared service returns raw AP-REP only.

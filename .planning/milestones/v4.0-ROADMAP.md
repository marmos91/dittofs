# Milestone Archive: v4.0 BlockStore Unification Refactor

**Status:** SHIPPED 2026-03-11
**Phases:** 41-49
**Total Plans:** 24

## Overview

Complete storage layer refactor replacing confusing PayloadService/Cache/DirectWrite layers with clean two-tier block store model (Local + Remote). Each share gets its own BlockStore with isolated local storage and ref-counted shared remote stores. Includes L1 read cache with adaptive prefetcher and platform-aware auto-configuration.

## Milestone Summary

**Key Decisions:**

- Two-tier block store model (Local + Remote) replacing PayloadService/Cache/DirectWrite
- Per-share block stores with isolated local paths and ref-counted shared remote stores
- BlockStoreConfig with Kind discriminator instead of PayloadStoreConfig
- Package restructure: pkg/blockstore/ absorbing pkg/cache/ and pkg/payload/
- Platform-aware sysinfo detection for auto-deduced defaults
- Copy-on-read semantics for L1 cache to prevent corruption

**Issues Resolved:**

- Confusing three-layer storage architecture simplified to two tiers
- Global PayloadService replaced with per-share isolation
- Dead code removed (DirectWriteStore, fs payload store)
- Auto-configuration prevents OOM on resource-constrained systems

**Issues Deferred:**

- Disk cache tier (memory -> SSD demotion) — deferred to future optimization
- GOMEMLIMIT enforcement — formulas defined but not wired to runtime

**Technical Debt Incurred:**

- GORM migration code retains old payload_stores table references (migrates FROM those names)
- Type aliases (PayloadStore = BlockStore) kept for backward compatibility

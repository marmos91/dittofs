**SMB handlers (read.go, write.go, close.go, flush.go, handler.go):**

1. Each SMB handler currently calls `h.Registry.GetBlockStore()`. Replace with `h.Registry.GetBlockStoreForHandle(ctx, handle)` where `handle` is the metadata handle from the open file state.

In SMB handlers, the open file typically has a `MetadataHandle` field (the NFS-style file handle encoding the share name). Find the handle for each call site:
- `read.go`: The read handler has access to the open file's MetadataHandle. Use `metadata.FileHandle(openFile.MetadataHandle)`.
- `write.go`: Same pattern.
- `close.go`: Has 3 call sites. Each has access to the open file or its metadata handle.
- `flush.go`: Has access to the open file.
- `handler.go`: Called during file operations, has access to the open file.

2. For each call site, the pattern change is:
```go
// OLD:
blockStore := h.Registry.GetBlockStore()
if blockStore == nil {
    // error handling
}

// NEW:
blockStore, err := h.Registry.GetBlockStoreForHandle(ctx, metadata.FileHandle(openFile.MetadataHandle))
if err != nil {
    // error handling - translate to SMB STATUS_INTERNAL_ERROR
}
```

Where `ctx` is already available (it's Go's context.Context from the request or background) and `openFile.MetadataHandle` is the file handle. If MetadataHandle is `[]byte`, wrap it as `metadata.FileHandle(openFile.MetadataHandle)`.

3. For nil-safety: if `openFile` or `MetadataHandle` could be empty, add a guard. In practice these are always set for files that have a BlockStore.

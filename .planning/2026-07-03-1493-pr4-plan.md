## What PR4 is

Finish the epic: migrate all remaining standalone `cas/<hash>` data (local per-chunk files +
remote objects + standalone locators) into packed blocks, then delete the entire standalone-CAS
code path, apply the final chunk/block naming model, and do the final docs pass.

## Scope

1. **Automatic, blocking, resumable cas→blocks migration** at share block-store startup.
2. **Delete the standalone/CAS path end to end** (remote hash-keyed interface, legacy mirror
   writer, standalone read branch, cas/ GC sweeps, local per-chunk chunk files, migrate-to-cas
   tool, dead fields/tests).
3. **Final naming sweep**: `BlockRef`→`ChunkRef`, `BlockState`→`ChunkSyncState`,
   `BlockChunkCommit`→`ChunkCommit`, `ErrCASContentMismatch`→`ErrChunkContentMismatch`, helpers.
4. **Docs pass** across `docs/internals/` + `docs/guide/`; regenerate `cli.md` (migrate-to-cas
   command removal changes the command tree).

## Non-goals (explicitly deferred)

- #1525 orphan packed-block reconcile sweep (PR5). Migration is designed to add at most one
  orphan block object per crash — same class/window as the live carver — and no record leaks.
- Local logblob byte reclaim, restart-seed of uncarved local-only chunks (PR5).
- #1487 GC compaction, #1488 chunk-range refetch, #1489 remote manifests, #1490 scrubber.
- Renaming DB table/column names or JSON wire tags (schema stays; Go identifiers only).
- `ObjectID` keeps its name (file-level Merkle root — not a chunk identifier; load-bearing
  for dedup via `FindByObjectID`).

## Locked decisions

- **No sentinel file, no journal, no config knobs.** The data is the state: migration triggers
  on (a) legacy per-chunk files present under the FSStore baseDir, (b) any synced hash whose
  locator is standalone (`BlockID == ""`), (c) any object left under remote `cas/`. Detection
  costs are one local dir probe + one O(synced-set) metadata scan + one remote LIST page per
  share per boot. Re-running always converges; killed migrations resume by construction.
- **Migration lives in the engine** (`pkg/block/engine`), runs synchronously inside
  `Store.Start` before the share serves. This inherits per-share block stores, ref-counted
  remote lifecycles, and the carver's seal/commit plumbing with zero new runtime wiring.
- **Atomic per-block commit including locator rewrite.** `metadata.Transaction` gains
  `SyncedHashStore` (all 4 backends implement it on their tx types, storetest-conformed), and
  `DefaultCommitBlock` moves its `MarkSynced` loop inside the transaction. This deletes
  `ErrCommitPartial` + the carver's retry loop, shrinks the #1525 orphan window for the live
  path, and is what makes migration re-runs leak-free: a block record is durable if and only
  if all its chunks' locators point at it.
- **Legacy standalone reader survives only as migration-scoped code:** `pkg/block/legacy_cas.go`
  (key helpers) + `legacy_cas_migration.go` files on the s3/memory backends and the
  compression/encryption decorators, exposing exactly
  `WalkLegacyChunks` / `ReadLegacyChunkVerified` / `DeleteLegacyChunk`. Everything else dies.
- **migrate-to-cas (.blk→cas offline tool) is deleted.** It writes standalone cas objects via
  the removed Put-by-hash surface. Pre-CAS `.blk` layouts must migrate with dittofs ≤ v0.21
  first; the boot guard keeps refusing legacy layouts with that message (exit 78 preserved).
- **Streaming bounds:** migration batches plaintext up to `BlockCarveBytes` (~16 MiB) exactly
  like the live carver; no whole-namespace slurp. Chunk bytes come from the local tier when
  present (no download), else one legacy verified GET.

## Migration design

Per share, inside `engine.Store.Start`, before the syncer loops start serving:

**Phase L — local per-chunk files → logblob.** If `<baseDir>/blocks/<xx>/<yy>/<hash>` files
exist (pre-flip layout): read each, verify `blake3(bytes)==hash` (skip+Error on mismatch),
`StoreChunk` through the logblob path (HasChunk dedups), delete the file, prune empty shard
dirs. Idempotent: a crash re-imports the remainder; dedup makes re-imports no-ops. After this,
the FSStore legacy per-chunk read/write fallbacks are deleted and logblob+index become
mandatory (constructor-enforced).

**Phase R — standalone locators → packed blocks.** `EnumerateSynced`; every hash whose locator
is standalone joins the migration set. Accumulate batches up to `BlockCarveBytes`:
- bytes: local `Get(hash)` when `Has(hash)`, else `ReadLegacyChunkVerified(hash)` (BLAKE3
  fail-closed, exactly the old `ReadBlockVerified` semantics). Missing in both places =
  pre-existing data loss: log Error with the hash, drop the marker, count it, continue.
- seal via `ChunkSealer.SealChunk` → `blockcodec.Builder.Add` (same as carver).
- flush: `PutBlock` → single transaction {idempotency guard, `PutBlockRecord`
  (LiveChunkCount = len(batch)), `PutLocalLocation` for locally-present chunks only,
  per chunk `DeleteSynced`+`MarkSynced(blockLocator)`} → best-effort `DeleteLegacyChunk`
  per migrated cas object (failures left to Phase P).

Crash windows: before `PutBlock` → nothing recorded, re-run repacks. Between `PutBlock` and
the tx → one orphan `blocks/` object (live-carver class, PR5 reconcile), re-run repacks into
a fresh block. After the tx → locators already point at the new block; the leftover cas
objects are unreferenced and Phase P removes them. No partially-pointed block records can
exist, so no `LiveChunkCount` leaks.

**Phase P — purge.** `WalkLegacyChunks` over the remote `cas/` namespace; delete every
remaining object. After Phase R none of them is referenced (each was either migrated, or was
never marked synced — the pre-flip Put-then-Mark crash orphans the old sweepByWalk existed
for). Also idempotent.

**Byte-identity guarantee:** legacy reads are BLAKE3-verified before packing; the new read
path re-verifies `blake3(plaintext)==hash` fail-closed on every block-range read; tests
assert byte-identical read-back post-migration.

## File-by-file work plan

### A. Metadata: transactional synced-hash ops (prerequisite)
- `pkg/metadata/store.go` — `Transaction` embeds `SyncedHashStore`.
- `pkg/metadata/store/{memory,badger,sqlite,postgres}/…` — implement
  IsSynced/MarkSynced/GetLocator/DeleteSynced on the tx types (reuse store-level bodies).
- `pkg/metadata/block_record_store.go` — `DefaultCommitBlock`: MarkSynced moves into the tx;
  skip `PutLocalLocation` for zero-valued local locations (migrated chunks have none);
  delete `ErrCommitPartial`.
- `pkg/metadata/storetest/` — conformance: tx-level synced-hash ops (visibility, rollback),
  atomic CommitBlock (locator + record commit or neither), locator overwrite via
  DeleteSynced+MarkSynced in one tx.
- `pkg/block/engine/carver.go` — drop the ErrCommitPartial retry loop.

### B. Migration
- `pkg/block/legacy_cas.go` (new) — cas-key helpers, migration-scoped, documented for
  deletion with the migration in a future release.
- `pkg/block/remote/remote.go` — `LegacyCASStore` capability (Walk/ReadVerified/Delete).
- `pkg/block/remote/s3/legacy_cas_migration.go`, `pkg/block/remote/memory/…`,
  compression + encryption decorator legacy files — implementations/forwards (read unseals).
- `pkg/block/engine/legacy_migration.go` (new) — Phases R+P as above; called from `Start`.
- `pkg/block/local/fs/legacy_migration.go` (new) — Phase L; invoked via the engine.
- Tests: fixture generator writes pre-flip state (sealed standalone objects at cas keys +
  standalone synced markers + legacy per-chunk local files) using test-only helpers; assert
  byte-identical reads, locators rewritten, `cas/` empty, LiveChunkCount correct, re-run
  no-op, kill-after-N-chunks resume converges without record leaks.

### C. Deletion (after B lands in-branch, same PR)
- `pkg/block/blockstore.go` — `Store` stays as the LOCAL tier contract only; remote loses it.
- `pkg/block/remote/remote.go` — `RemoteStore` = RemoteBlockStore + ChunkSealer + ChunkReader
  folded in + Close/Healthcheck; delete hash-keyed embed, `ReadBlockVerified`,
  `ErrChunkReadUnsupported` capability dance.
- `pkg/block/remote/s3/store.go`, `remote/memory` — delete Put/Get/GetRange/Has/Delete/Head/
  Walk-by-hash + ReadBlockVerified.
- compression/encryption decorators — delete hash-keyed wrappers.
- `pkg/block/types.go` — delete FormatCASKey/ParseCASKey/CASKeyPrefix (move minimal helpers
  into `legacy_cas.go`); delete vestigial `FileChunk.BlockStoreKey` (+ scan sites in
  sqlite/postgres `objects.go`; columns stay).
- `pkg/block/locator.go` — delete `IsStandalone`; standalone docs rewritten.
- `pkg/block/engine/fetch.go` — `dispatchRemoteFetch` standalone branch dies; a synced hash
  without a block locator is drift → fail closed.
- `pkg/block/engine/syncer.go` — delete mirrorOnce/mirrorChunk/standalone MarkSynced +
  `rerouteCarveMiss` (carve miss on local index = drop + Error, parallel to vanished-bytes).
- `pkg/block/engine/gc.go` — delete `sweepByWalk` (cas namespace walk); gc_sweep_index.go
  drops the standalone Delete fallthrough (BlockReclaimer is the only reclaim).
- `pkg/block/local/fs/chunkstore.go` — legacy per-chunk StoreChunk/ReadChunk branches die;
  logblob+index mandatory; per-chunk LRU/eviction/diskUsed remnants removed
  (**watch #1527 fix/1527-disk-used-underflow for rebase**).
- `cmd/dfs/commands/migrate_to_cas.go`, `pkg/block/migrate/migrate_to_cas.go` (+tests),
  `NewFSStoreForMigration`, sentinel consts — deleted; `start.go` boot guard keeps refusing
  legacy `.blk` layouts (exit 78) with "migrate with dittofs ≤ v0.21".
- Tests: `engine_dualread_test.go`, `chunkstore_test.go`, `gc_test.go`,
  `gc_sweep_index_test.go`, `types_test.go` key round-trips,
  `pkg/metadata/storetest/file_block_ops.go`, `runtime/snapshot_lifecycle_test.go`, ~58 files.
